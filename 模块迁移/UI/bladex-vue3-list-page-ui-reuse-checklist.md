# BladeX Vue3 列表页 UI 复用清单

本文档用于把当前 RuoYi 项目中“入驻管理 / 园区管理 / 财务管理 /合同管理/ 企业服务”等页面的统一 UI 风格抽成一份可复用清单，方便后续迁移到 BladeX `saber3` Vue3 架构时保持页面观感、交互结构和开发习惯一致。

这份清单不是业务迁移清单，而是“页面 UI 骨架复用清单”。你后面迁任何列表页，都可以先拿这份文档对照，判断该页面是不是标准列表页、要复用哪些组件、哪里允许做特例。

## 1. 适用范围

适合以下类型页面：所有一级子菜单

- 园区管理列表页
- 楼栋管理列表页
- 入驻商机列表页
- 客户列表页
- 缴费管理列表页
- 物业服务列表页
- 商户管理列表页
- 合同列表页

不完全适合以下类型页面：

- 楼层管理这种“主表 + 展开子表”的复杂表格页
- 租控管理这种看板页
- 多页签复合详情页
- 流程设计器、表单设计器

首页工作台不适合直接套“标准列表页模板”，但它必须纳入同一套 UI 规范，不能单独做成廉价占位页。

## 2. 当前 UI 风格结论

### 2.1 RuoYi 现状

当前源端页面风格并不是每个页面完全手写一套，而是复用了几类公共构件：

- 标题区：`BusinessPageHeading`
- 统计卡片区：`BusinessSummaryCards`
- 主列表：`STable`
- 查询区样式类：
  - `.table-page-search-wrapper`
  - `.table-page-search-submitButtons`
  - `.filter-search-btn`
  - `.filter-reset-btn`
- 操作按钮区：
  - `.table-operator`
  - `.table-action-btn`

典型页面：

- [ParkList.vue](/Users/jiakangli/桌面/ics-park/ruoyi-ui/src/views/business/ParkList.vue)
- [DeclareService.vue](/Users/jiakangli/桌面/ics-park/ruoyi-ui/src/views/business/DeclareService.vue)
- [ApprovalProjectList.vue](/Users/jiakangli/桌面/ics-park/ruoyi-ui/src/views/business/ApprovalProjectList.vue)

### 2.2 BladeX Vue3 现状

BladeX `saber3` 当前页风格和 RuoYi 不完全同构，但也已经有对应复用基础：

- 外层容器：`basic-container`
  - [basic-container/main.vue](/Users/jiakangli/Desktop/bladex-park/saber3/src/components/basic-container/main.vue)
- 统计卡片：`stat-cards`
  - [stat-cards/main.vue](/Users/jiakangli/Desktop/bladex-park/saber3/src/components/stat-cards/main.vue)
- 主列表多使用 `avue-crud`
  - [park.vue](/Users/jiakangli/Desktop/bladex-park/saber3/src/views/park/park.vue)
  - [payment.vue](/Users/jiakangli/Desktop/bladex-park/saber3/src/views/finance/payment.vue)

所以迁移时不要机械照搬 `STable`，而要把“RuoYi 的统一视觉结构”迁成“BladeX Vue3 的统一页面模板”。

### 2.3 导航布局现状

BladeX `saber3` 当前已经具备“顶部主菜单 + 左侧业务菜单”的基础能力：

- 顶部横向一级菜单：
  - [top-menu.vue](/Users/jiakangli/Desktop/bladex-park/saber3/src/page/index/top/top-menu.vue)
- 左侧纵向菜单：
  - [sidebar/index.vue](/Users/jiakangli/Desktop/bladex-park/saber3/src/page/index/sidebar/index.vue)
- 全局设置：
  - [website.js](/Users/jiakangli/Desktop/bladex-park/saber3/src/config/website.js)

源码里默认 `setting.sidebar = 'vertical'`，但同时存在顶部 `top-menu` 作为主导航入口。对你这个园区项目，推荐明确固化成：

- 一级菜单：顶部横向导航
- 二级菜单：左侧纵向导航
- 页面内容：右侧主工作区

不要把一级和二级都塞进左侧树菜单里，否则会和你现在这套迁移目标不一致。

### 2.4 颜色体系现状

BladeX `saber3` 已有颜色和主题能力：

- 主色存储：
  - `colorName`
  - 默认值：`#2C77F1`
- 主题存储：
  - `themeName`
  - 默认值：`theme-go`
- 主题文件目录：
  - `saber3/src/styles/theme/*.scss`

但当前清单原版还没有把“园区业务页面该固定哪套颜色层级”写死，所以这次要补成可执行规范：迁移时不能只依赖用户在设置面板里手动选色，业务页本身也要按统一 token 设计。

## 3. 标准列表页骨架

标准列表页建议统一采用这 6 段结构：

1. 外层容器
2. 页面标题
3. 统计卡片
4. 查询筛选区
5. 主操作区
6. 主列表区

推荐结构：

```text
basic-container
  -> page-header
  -> stat-cards
  -> search-panel
  -> toolbar
  -> avue-crud / table
  -> dialog / drawer / detail
```

## 3.3 首页工作台 UI 规范

首页必须单独作为“工作台首页”处理，不能按普通列表页糊一个占位页面。

### 当前问题

从 BladeX 现有首页实现看：

- [home.vue](/Users/jiakangli/Desktop/bladex-park/saber3/src/views/home.vue)

它现在更像“可点击的功能骨架页”，而不是成熟后台首页，主要问题有：

- Banner 区过于简陋，没有形成稳定视觉中心。
- 指标卡片像占位卡，而不是业务指标卡。
- 内容区块之间层级弱，像拼接。
- 缺少原 RuoYi 首页那种“运营工作台”气质。
- 快捷入口、招商闭环、待办、政策、企业展示之间没有统一节奏。
- 页面虽然能用，但不够像正式产品首页。

### 首页应继承的源端气质

RuoYi 原首页：

- [Analysis.vue](/Users/jiakangli/桌面/ics-park/ruoyi-ui/src/views/dashboard/Analysis.vue)

它的核心不是“炫”，而是这几个特征：

- 首屏就是工作台，不是空容器。
- 上方指标卡一眼扫全局。
- 中间大 Banner 提供统一业务语义。
- 左主右辅布局明显。
- 核心入口、政策通知、快捷入口、企业展示、待办任务层次分明。
- 每个板块既能独立看，也能形成整体节奏。

### 首页布局规范

首页推荐固定成下面这个结构：

```text
首页工作区
  -> 顶部指标卡
  -> 主体双栏布局
     -> 左侧主栏
        -> Banner
        -> 招商闭环入口
        -> 政策通知 + 快捷入口
        -> 入驻企业展示
     -> 右侧辅栏
        -> 日历 / 日程
        -> 待办提醒
```

迁移要求：

- [ ] 首页不要复用普通列表页的“标题 + 查询 + 表格”结构。
- [ ] 首页采用工作台专属布局。
- [ ] 顶部指标卡独立成一行。
- [ ] 主体区必须是“左主右辅”。
- [ ] 左侧承担核心业务信息。
- [ ] 右侧承担提醒、轻量辅助信息。

### 首页 Banner 规范

- [ ] Banner 必须是首页视觉中心。
- [ ] Banner 不能只是纯文字块。
- [ ] Banner 要有明确的标题、描述、业务语义。
- [ ] Banner 要有图片或视觉背景支撑。
- [ ] Banner 高度稳定，不要忽高忽低。
- [ ] Banner 按业务页风格，保持正式、清爽、可信。

不建议：

- [ ] 不要做成营销官网 Hero。
- [ ] 不要只放两个按钮和一段文案就结束。
- [ ] 不要用过重的渐变、过重阴影、夸张插画。

### 首页指标卡规范

- [ ] 指标卡一行展示 4 到 5 张。
- [ ] 每张卡包含图标、标题、数值、轻说明。
- [ ] 指标卡要能点击进入对应业务模块。
- [ ] 图标可以有轻度色块，但不要五颜六色失控。
- [ ] 数值要比普通列表页卡片更醒目。

建议字段：

- `title`
- `value`
- `desc`
- `icon`
- `path`
- `tone`

### 首页区块规范

首页区块建议统一采用：

- 白底
- 轻边框或轻阴影
- 小圆角
- 上方标题 + 右上角次操作

统一区块包括：

- [ ] 招商闭环入口
- [ ] 园区政策通知
- [ ] 快捷入口
- [ ] 入驻企业展示
- [ ] 待办提醒
- [ ] 日历 / 日程

每个区块必须有：

- [ ] 清晰标题
- [ ] 简短辅助说明或次要操作
- [ ] 稳定内边距
- [ ] 稳定高度或最小高度

### 首页快捷入口规范

- [ ] 快捷入口必须像工具入口，不像普通按钮列表。
- [ ] 每项包含图标和标题。
- [ ] 图标区风格统一。
- [ ] 未迁完模块要禁用或给占位说明。
- [ ] 不允许点击后 404。

### 首页政策通知规范

- [ ] 每条政策展示标题和日期。
- [ ] 样式要偏信息列表，不是卡片瀑布流。
- [ ] 政策列表高度稳定。
- [ ] 没有数据时显示正式空状态。

### 首页企业展示规范

- [ ] 企业展示不应只是三条占位文本。
- [ ] 至少要具备企业简称、企业名、行业。
- [ ] 如果做轮播或滚动，滚动节奏要轻。
- [ ] 空数据时要像业务空状态，不像开发占位。

### 首页待办提醒规范

- [ ] 待办提醒是右侧最重要区块之一。
- [ ] 每条提醒有数量、标题、说明。
- [ ] 数量突出，但不要做成大屏数字牌。
- [ ] 点击能跳到对应业务页。

### 首页日历 / 日程规范

- [ ] 日历作为辅助区块放右侧。
- [ ] 样式轻，不抢主内容。
- [ ] 若当前没有真实日程能力，文案要克制。
- [ ] 不要把“待开发”做成大面积提示块。

### 首页颜色规范

首页必须遵守整体 token，但允许有比列表页稍强一点的视觉层次。

- [ ] 主背景仍以浅灰白工作区为主。
- [ ] 卡片和区块保持白底。
- [ ] Banner 可使用蓝色系品牌渐变。
- [ ] 指标卡的强调色必须收敛在统一色板内。
- [ ] 不允许出现紫色主导、彩虹配色、深色大面积压顶。

首页推荐额外 token：

```text
--park-home-hero-start: #1167E8
--park-home-hero-end: #D8E8FF
--park-home-panel-shadow: 0 4px 16px rgba(30, 64, 120, 0.06)
--park-home-panel-shadow-hover: 0 8px 18px rgba(22, 119, 255, 0.12)
```

### 首页交互规范

- [ ] 首页首屏所有核心入口必须可点击。
- [ ] 点击跳转逻辑清晰，不绕路。
- [ ] 任何未接入数据的区块都不能只写“待接入”糊弄过去。
- [ ] 允许空状态，但空状态也要像产品态。

### 首页禁止事项

- [ ] 不要复用 `wel/dashboard.vue` 这种演示页风格。
- [ ] 不要做成一堆等宽按钮拼盘。
- [ ] 不要把首页做成纯占位导航页。
- [ ] 不要让首页比业务列表页还粗糙。
- [ ] 不要出现模块区块高低失衡、边距紊乱、信息密度忽大忽小。

### 首页在 BladeX Vue3 的落地建议

首页迁移到 Saber3 时，建议按“工作台专属页面”处理，不要直接套通用列表页。

推荐页面结构：

```text
top-menu
  -> sidebar
    -> home.vue
       -> basic-container
          -> HomeMetricCards
          -> HomeMainLayout
             -> HomeHeroBanner
             -> HomeInvestmentPanel
             -> HomePolicyPanel
             -> HomeQuickEntryPanel
             -> HomeEnterprisePanel
             -> HomeTodoPanel
             -> HomeCalendarPanel
```

组件复用建议：

- [ ] 外层继续使用 `basic-container`，保持和 BladeX 业务页容器一致。
- [ ] 顶部指标卡单独抽 `HomeMetricCards`，不要和列表页统计卡完全混写。
- [ ] Banner 单独抽 `HomeHeroBanner`，不要把 Banner 文案塞进普通卡片头部。
- [ ] 快捷入口单独抽 `HomeQuickEntryPanel`，统一图标、间距、hover 和禁用态。
- [ ] 政策通知、企业展示、待办提醒、日历建议都抽成独立 panel 组件。
- [ ] 每个 panel 统一收口到“标题 + 内容 + 右上角操作”结构。
- [ ] 首页组件只承担展示与跳转，接口聚合逻辑尽量收敛到 `home.vue` 或专属 `api/home.js`。

样式落地建议：

- [ ] 首页整体背景沿用系统工作区背景，不额外换深色底。
- [ ] 首页区块圆角保持 8px 以内，和现有 BladeX 业务卡片统一。
- [ ] 首页区块阴影必须轻，不要出现漂浮感很重的大阴影。
- [ ] 首页标题字号、正文字号、辅助文案字号要有明确层级，不要全部一样大。
- [ ] 首屏高度要兼顾 1440 宽屏和常规笔记本，不要让 Banner 把首屏全部占满。

数据装配建议：

- [ ] 顶部指标卡接口聚合园区总量、企业总量、待办总量、财务提醒等核心摘要。
- [ ] Banner 区只放一条主叙事，不要并列塞多个业务主题。
- [ ] 快捷入口按高频动作排序，优先放“园区档案、企业入驻、合同、审批、缴费”等。
- [ ] 待办提醒优先接审批、合同到期、账单逾期、服务工单等真实任务。
- [ ] 没有接入真实接口前，先做稳定 mock 结构，避免 UI 做完后再次拆布局。

### 首页 UI 校验清单

首页迁移完成后，至少逐项核对以下内容：

- [ ] 一级菜单仍在顶部横向展示，二级菜单仍在左侧纵向展示，首页没有自建第二套路由导航。
- [ ] 首屏进入后，用户第一眼看到的是“指标卡 + Banner + 关键工作区块”，不是空白容器。
- [ ] 顶部指标卡一行 4 到 5 张，尺寸统一，数值、标题、图标基线统一。
- [ ] Banner 是首屏视觉中心，但不会挤压下面主要业务区块出首屏。
- [ ] 左主右辅结构稳定，左侧明显比右侧更重，右侧只是辅助提醒区。
- [ ] 招商闭环、政策通知、快捷入口、企业展示、待办提醒、日历这几个区块节奏统一。
- [ ] 快捷入口看起来像工具入口矩阵，不像一排普通按钮。
- [ ] 政策通知和待办提醒的数据列表行高统一，不出现一边稀疏一边拥挤。
- [ ] 空状态具备插画或说明文案，不出现“待开发”“暂无接入”这类粗糙占位。
- [ ] 所有核心入口点击后都有明确跳转落点，不出现死链、404、空白页。
- [ ] 首页颜色仍然服从统一 token，不出现首页一套色、列表页另一套色的割裂。
- [ ] 首页风格明显优于默认演示页，不会让人感觉只是把 `wel/dashboard.vue` 改了几块文案。

## 3.1 导航布局复用清单

这是页面迁移时必须一起遵守的外层壳子规则。

### 一级菜单

- [ ] 一级菜单固定放在顶部横向导航。
- [ ] 一级菜单只承载主业务域：
- [ ] 首页
- [ ] 园区管理
- [ ] 入驻管理
- [ ] 合同管理
- [ ] 企业服务
- [ ] 财务管理
- [ ] 一级菜单数量控制在可扫描范围，不要无限展开。

### 二级菜单

- [ ] 二级菜单固定放在左侧纵向导航。
- [ ] 左侧只展示当前一级菜单下的业务菜单。
- [ ] 点击顶部一级菜单后，左侧菜单随之切换。
- [ ] 左侧菜单展开规则统一。
- [ ] 当前激活菜单高亮统一。

### 页面工作区

- [ ] 内容区固定在右侧主区域。
- [ ] 页面不要再自建第二套顶部页签导航替代一级菜单。
- [ ] 页面内部页签只用于同一业务页内的子视图切换。

### BladeX 落地建议

- [ ] 顶部菜单继续用 `top-menu`
- [ ] 左侧菜单继续用 `sidebar`
- [ ] 用 `GetTopMenu` 负责一级菜单
- [ ] 用 `GetMenu(topMenuId)` 负责加载当前一级菜单下的左侧菜单
- [ ] 对园区项目不要让用户随意切成“纯左侧纵向一级菜单”模式

## 3.2 颜色配置复用清单

### 颜色规范目标

- [ ] 页面颜色必须可配置，但业务默认色要固定。
- [ ] 一级导航、二级导航、按钮、状态、卡片、边框使用统一层级。
- [ ] 不允许每个业务页面各写一套主色。
- [ ] 不允许出现明显偏紫、偏营销、偏炫彩的色彩漂移。

### 建议默认主题

结合当前 BladeX 默认值和你现有园区业务页风格，建议园区项目默认采用：

- 主色：`#2C77F1`
- 主题：`theme-go` 或基于 `theme-go` 做园区项目轻定制

原因：

- 蓝色主色和你现有 RuoYi 业务页最接近
- 适合后台运营、资产、审批、财务类系统
- 和白底卡片、浅边框组合稳定

### 建议颜色 token

迁移时建议至少固定这些 token：

```text
--park-color-primary: #2C77F1
--park-color-primary-light: #F0F6FF
--park-color-success: #23B26D
--park-color-warning: #F5A623
--park-color-danger: #E34D59
--park-color-text-main: #1F2D3D
--park-color-text-secondary: #6F7B8A
--park-color-border: #E8EDF3
--park-color-card-bg: #FFFFFF
--park-color-page-bg: #F5F7FA
--park-color-sidebar-bg: #FFFFFF
--park-color-sidebar-active-bg: #F0F6FF
--park-color-topbar-bg: #FFFFFF
```

### 颜色使用规则

- [ ] 主按钮、激活态、链接强调使用主色。
- [ ] 卡片背景统一白色。
- [ ] 页面背景统一浅灰白。
- [ ] 边框统一浅灰蓝。
- [ ] 标题正文用深色文本。
- [ ] 辅助说明用次级灰蓝文本。
- [ ] 成功、警告、危险状态用固定语义色。

### 状态颜色建议

- [ ] 启用 / 已完成 / 已缴 / 正常：绿色
- [ ] 待处理 / 草稿 / 进行中：蓝色
- [ ] 预警 / 即将到期 / 部分缴纳：橙色
- [ ] 逾期 / 停用 / 驳回 / 删除风险：红色
- [ ] 中性 / 未知 / 未开始：灰色

### 不建议做法

- [ ] 不要每页自己硬编码主色
- [ ] 不要让顶部菜单和左侧菜单使用两套完全不同的主色
- [ ] 不要为了区分模块给每个一级菜单换一套主题色
- [ ] 不要使用高饱和渐变做后台页面主色

## 4. 页面骨架复用清单

### 4.1 外层容器

目标：

- [ ] 列表页统一放在 `basic-container` 内。
- [ ] 页面内容不要直接裸挂在根节点下。
- [ ] 内边距保持稳定。
- [ ] 页面主体和外围布局解耦。

BladeX 建议：

- 统一使用 `basic-container`
- 默认 `radius` 不要太大
- 业务页保持白底卡片容器感
- 页面背景建议跟随统一 token，而不是每页单独设背景色

### 4.2 页面标题

RuoYi 风格特征：

- 单行标题
- 下方一条浅边框分隔
- 标题偏业务页，不是大屏 Hero

迁移要求：

- [ ] 每个业务列表页都有固定标题区。
- [ ] 标题与筛选区之间留 12px 到 16px 间距。
- [ ] 标题字体不要过大，保持工作台风格。
- [ ] 不要在列表页使用营销化 Hero 标题。
- [ ] 标题颜色统一使用主文本色 token。

推荐抽象：

- `PageSectionHeader`

建议字段：

- `title`
- `subtitle` 可选
- `extra` 可选

### 4.3 统计卡片

RuoYi 风格特征：

- 位于标题下方
- 一行 4 到 5 张
- 白底、浅边框、小圆角
- 标签在上，数值在下
- 数值明显大于标签

迁移要求：

- [ ] 统计卡片放在列表主表格之前。
- [ ] 卡片高度保持统一。
- [ ] 一组卡片语义一致，避免混杂不同层级指标。
- [ ] 移动端自动降为 2 列。
- [ ] 卡片背景、边框、标题色、数字色统一走 token。

BladeX 建议：

- 优先复用现有 `stat-cards`
- 如果 `stat-cards` 表现不够接近源端，可以做一层园区业务包装组件：
  - `BusinessStatCards`

建议统一字段：

- `key`
- `label`
- `value`
- `hint` 可选
- `status` 可选

### 4.4 查询筛选区

RuoYi 风格特征：

- 放在表格上方
- 使用 inline form
- 标签左、输入右
- 同一行多个字段
- “查询 / 重置”作为固定组合

迁移要求：

- [ ] 查询区统一放在列表上方。
- [ ] 常规搜索项优先一行 3 到 4 个。
- [ ] 输入框、下拉、日期范围高度统一。
- [ ] 查询按钮和重置按钮位置稳定。
- [ ] 高级筛选项可折叠，不要默认铺满首屏。
- [ ] 查询区不要额外染色，保持白底工作区风格。

BladeX Vue3 建议：

- 用 `avue-crud` 自带 `search` 能力时，统一 option 配置风格。
- 如果页面查询条件复杂，独立抽一个 `SearchPanel`，不要把所有逻辑塞进 `option.search`。

建议统一组件：

- `BusinessSearchPanel`

建议统一行为：

- [ ] 支持回车查询。
- [ ] 重置后回到第一页。
- [ ] 刷新列表统一调用同一个 reload 方法。
- [ ] 路由 query 与筛选项要能互相回填时，统一封装。

### 4.5 主操作区

RuoYi 风格特征：

- 通常在查询区下方
- 放新增、删除、导出、行业规则、刷新等按钮
- 左对齐为主
- 多个动作间距统一

迁移要求：

- [ ] 列表主操作区和行内操作分开。
- [ ] 批量动作放左侧工具栏，不要混进单行操作列。
- [ ] 只有该页面的主动作才放工具栏。
- [ ] 危险动作和普通动作视觉区分清楚。
- [ ] 工具栏按钮颜色遵守统一语义色。

BladeX 建议：

- `avue-crud` 可用 `menu-left` 插槽承载主工具栏。
- 页面若需要独立布局，也可以抽 `BusinessToolbar`。

### 4.6 主列表区

这是复用最关键的部分。

RuoYi 当前主列表多数使用 `STable`，特点是：

- 远程分页
- 统一 `pageNum` / `pageSize`
- 统一 `refresh(true)`
- 统一接收 `rows + total + pageNum`

BladeX Vue3 当前主列表多数使用 `avue-crud`，特点是：

- `v-model:page`
- `on-load`
- `search-change`
- `refresh-change`
- `current-change`
- `size-change`

迁移要求：

- [ ] BladeX 统一使用一种主列表模式。
- [ ] 标准 CRUD 页优先走 `avue-crud`。
- [ ] 极复杂表格才考虑退回 `el-table` 手写。
- [ ] 所有列表页分页交互保持一致。
- [ ] 所有列表页空状态风格一致。
- [ ] 所有列表页 loading 风格一致。
- [ ] 表格头部、边框、hover、选中态遵守统一色板。

建议判断原则：

- 简单列表页：优先 `avue-crud`
- 带展开行、嵌套子表、复杂单元格编排：可手写 `el-table`
- 详情抽屉里的子表：直接 `el-table` 即可，不强行套 `avue-crud`

## 5. 表格列风格复用清单

### 5.1 列宽策略

- [ ] 核心标识列设固定宽度。
- [ ] 操作列固定在右侧。
- [ ] 文本较长列启用省略或 tooltip。
- [ ] 金额、状态、日期列宽尽量稳定。

### 5.2 状态展示

RuoYi 里大量使用 tag。

迁移要求：

- [ ] 状态统一用 `el-tag` 展示。
- [ ] 相同状态码在不同页面颜色一致。
- [ ] 状态文案统一从字典或映射方法读取。
- [ ] 不要每个页面各写一套状态颜色。

建议抽象：

- `StatusTag`

输入：

- `status`
- `dic` 或 `type`

### 5.3 金额、面积、数量展示

- [ ] 金额统一两位小数。
- [ ] 面积统一保留业务要求的小数位。
- [ ] 数量统一显示单位。
- [ ] 空值统一显示 `-`。

建议抽公共方法：

- `formatMoney`
- `formatArea`
- `formatCount`
- `formatPercent`

### 5.4 链接列

RuoYi 很多主键或名称列可点击进入详情。

迁移要求：

- [ ] 列表里只有关键识别字段做链接。
- [ ] 链接颜色统一。
- [ ] 点击行为统一为详情抽屉或编辑页，不混乱。

## 6. 详情层复用清单

### 6.1 弹窗

适用于：

- 新增
- 编辑
- 简单确认

迁移要求：

- [ ] 表单弹窗宽度保持稳定。
- [ ] footer 按钮顺序统一。
- [ ] 保存中状态统一。
- [ ] 关闭时表单清理一致。

### 6.2 抽屉

适用于：

- 查看详情
- 复杂信息浏览
- 时间线、日志、子表联动

迁移要求：

- [ ] 详情抽屉宽度统一分级。
- [ ] 抽屉内优先用描述区 + 子表区 + 时间线区。
- [ ] 不把复杂编辑表单塞进纯详情抽屉。

### 6.3 子表格

RuoYi 中详情里的子表通常直接用 `a-table`。

迁移建议：

- [ ] BladeX 子表格优先用 `el-table`。
- [ ] 子表不强制复用 `avue-crud`。
- [ ] 子表只承担展示，不承担复杂 CRUD 时保持轻量。

## 7. 页面样式约束

### 7.1 整体风格

- [ ] 白底工作台风格，不做花哨大背景。
- [ ] 轻边框、轻阴影、小圆角。
- [ ] 以扫描效率优先，不做营销页布局。
- [ ] 一屏内信息密度适中。
- [ ] 页面默认主色统一，不因业务菜单切换而漂移。

### 7.2 间距

- [ ] 标题到卡片：12 到 16px
- [ ] 卡片到筛选区：12 到 16px
- [ ] 筛选区到工具栏：12px 左右
- [ ] 工具栏到表格：12px 左右

### 7.3 表单控件

- [ ] 输入、高度、边框风格统一。
- [ ] 查询按钮使用主按钮色。
- [ ] 重置按钮使用次按钮风格。
- [ ] 危险操作按钮只在必要时高亮。
- [ ] 激活态、focus 态边框统一使用主色 token。

### 7.5 导航视觉

- [ ] 顶部一级菜单使用横向布局。
- [ ] 顶部背景保持简洁，以白底为主。
- [ ] 一级菜单激活态使用主色强调。
- [ ] 左侧二级菜单使用纵向布局。
- [ ] 左侧菜单当前项高亮使用浅主色背景 + 主色文字。
- [ ] 左侧菜单背景保持稳定，不随页面频繁变化。
- [ ] 顶部与左侧导航视觉层级清晰，不互相抢重点。

### 7.4 响应式

- [ ] 桌面端优先保证多列筛选。
- [ ] 平板端自动减少列数。
- [ ] 手机端筛选项允许换行，不允许重叠。
- [ ] 统计卡片移动端至少降为两列。

## 9. 开发落地清单

### 9.1 迁移一个标准列表页时，先判断

- [ ] 是否有标题区
- [ ] 是否有统计卡片
- [ ] 是否是标准分页列表
- [ ] 是否需要主操作区
- [ ] 是否需要详情抽屉
- [ ] 是否需要弹窗编辑
- [ ] 是否需要子表
- [ ] 是否属于某个一级菜单下的标准二级页面
- [ ] 是否已经纳入“顶部一级 + 左侧二级”的导航体系

### 9.2 如果是标准页，优先复用

- [ ] `basic-container`
- [ ] 统一页头组件
- [ ] 统一统计卡片组件
- [ ] `avue-crud`
- [ ] 统一状态 tag
- [ ] 统一格式化方法

### 9.3 如果是复杂页，允许特例

- [ ] 楼层管理
- [ ] 租控管理
- [ ] 看板类页面
- [ ] 多层嵌套子表页面

但即便是特例页，也要尽量保留：

- [ ] 相同标题风格
- [ ] 相同卡片风格
- [ ] 相同按钮风格
- [ ] 相同状态展示风格

## 11. 校验清单

每迁一个 BladeX Vue3 列表页，都建议用这份 UI 校验清单过一遍：

- [ ] 页面外层使用统一容器。
- [ ] 标题样式统一。
- [ ] 统计卡片样式统一。
- [ ] 查询区布局统一。
- [ ] 查询按钮和重置按钮位置统一。
- [ ] 主操作区布局统一。
- [ ] 主列表分页行为统一。
- [ ] 状态 tag 颜色统一。
- [ ] 金额、面积、日期格式统一。
- [ ] 详情层交互统一。
- [ ] 手机端没有错位和重叠。
- [ ] 与同类页面放在一起时视觉不突兀。
- [ ] 顶部一级菜单是横向的。
- [ ] 左侧二级菜单是纵向的。
- [ ] 顶部一级菜单切换后，左侧菜单正确切换。
- [ ] 页面主色、边框色、背景色符合统一 token。
- [ ] 没有页面自己偷写一套色板。

## 12. 迁移时的执行建议

建议你后续迁页面时，先按这 3 步走：

1. 先判断这个页面是不是“标准列表页”
2. 如果是，就优先套这份 UI 复用清单
3. 如果不是，再把特殊结构单独写进该模块迁移清单

简单说，后面你迁 BladeX Vue3 时，不要再按“每页自己设计一套”来做，而是先把“标准列表页模板”固定住。这样园区管理、入驻管理、财务管理、审批中心迁出来之后，整体会像一个系统，不会像拼接出来的页面。
