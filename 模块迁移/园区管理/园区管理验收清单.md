# 园区管理验收清单

## 一、业务闭环

- [ ] 园区档案支持新增、编辑、删除、查看基础信息。
- [ ] 新增园区后，建筑管理可选择该园区。
- [ ] 删除园区前校验是否已关联建筑、楼层、房源、合同等业务数据。
- [ ] 建筑管理支持新增、编辑、删除、查看楼宇信息。
- [ ] 新增建筑时必须绑定所属园区。
- [ ] 建筑楼层数、建筑面积、可租面积等字段保存后能被楼层管理、租控管理使用。
- [ ] 楼层管理支持新增、编辑、删除楼层。
- [ ] 楼层必须绑定建筑，并通过建筑反向关联园区。
- [ ] 楼层下存在房源时，不允许直接删除楼层。
- [ ] 楼层编辑里可以新增房源。
- [ ] 楼层展开的房间名称后可以删除房源。
- [ ] 删除房源前校验是否已被合同、工单等业务引用。
- [ ] 租控管理能同步展示园区、建筑、楼层、房源数据。
- [ ] 房源新增、删除、状态变化后，楼层管理和租控管理数据一致。
- [ ] 房源出租、空置、预定、装修、停用状态统计正确回写到楼层统计。

## 二、数据流检查

- [ ] 园区主数据来源统一为 `ics_park`。
- [ ] 建筑主数据来源统一为 `ics_building`。
- [ ] 楼层主数据来源统一为 `ics_floor`。
- [ ] 房源主数据来源统一为 `ics_room`。
- [ ] 建筑数据必须包含 `park_id`。
- [ ] 楼层数据必须包含 `park_id`、`building_id`。
- [ ] 房源数据必须包含 `park_id`、`building_id`、`floor`。
- [ ] 新增房源时，后端根据建筑自动回填 `park_id`。
- [ ] 楼层统计中的房间数、已用面积、出租率来自房源数据聚合。
- [ ] 园区、建筑、楼层、房源名称在列表、详情、下拉选择中展示一致。
- [ ] 删除数据后，统计卡片、表格、下拉选项同步刷新。
- [ ] 前端展示字段与后端返回字段命名一致，不出现空字段、错字段。

## 三、接口一致性

- [ ] 园区档案统一使用 `/blade-park/park/*` 接口。
- [ ] 建筑管理统一使用 `/blade-park/building/*` 接口。
- [ ] 楼层管理统一使用 `/blade-park/floor/*` 接口。
- [ ] 租控房源管理统一使用 `/blade-park/room/*` 服务逻辑。
- [ ] 楼层管理里的新增、删除房源入口走 `/blade-park/floor/room/submit`、`/blade-park/floor/room/remove`。
- [ ] 楼层房源接口后端复用 `IRoomService`，不单独写重复业务逻辑。
- [ ] 同一业务操作不能出现两套不同校验规则。
- [ ] 前端 API 文件按模块维护，页面不直接硬编码 axios 请求。
- [ ] 所有删除接口都有后端引用校验。
- [ ] 所有提交接口都有后端必填校验。
- [ ] 接口返回统一使用 BladeX `R` 结构。
- [ ] 权限菜单与接口路径保持一致，避免页面按钮可见但接口无权限。

## 四、页面与交互

- [ ] 园区档案、建筑管理、楼层管理列表 UI 风格一致。
- [ ] 查询区、按钮区、表格区间距一致。
- [ ] 表格字段居中规则一致。
- [ ] 操作列按钮保持一行展示。
- [ ] 新增、编辑、删除按钮文案统一。
- [ ] 删除操作必须有确认弹窗。
- [ ] 保存成功、删除成功、校验失败都有明确提示。
- [ ] 表格刷新后分页、筛选条件不异常丢失。
- [ ] 空数据状态展示正常。
- [ ] 页面不出现右侧异常留白、按钮换行、字段遮挡。

## 五、重点回归场景

- [ ] 新增园区 -> 新增建筑 -> 新增楼层 -> 新增房源 -> 租控管理可见。
- [ ] 编辑园区名称 -> 建筑管理、楼层管理、租控管理展示同步更新。
- [ ] 编辑建筑名称 -> 楼层管理、租控管理展示同步更新。
- [ ] 编辑楼层面积 -> 新增房源面积校验使用最新楼层面积。
- [ ] 楼层新增房源 -> 楼层房间数、已用面积、出租率立即更新。
- [ ] 楼层删除房源 -> 楼层房间数、已用面积、出租率立即更新。
- [ ] 已被合同引用的房源删除失败，并提示原因。
- [ ] 有房源的楼层删除失败，并提示原因。
- [ ] 有楼层或房源的建筑删除失败，并提示原因。
- [ ] 有建筑的园区删除失败，并提示原因。

## 六、最终验收口径

- [ ] 园区、建筑、楼层、房源形成完整主数据链路。
- [ ] 所有页面看到的是同一套业务数据。
- [ ] 所有新增、编辑、删除都有后端校验兜底。
- [ ] 楼层管理和租控管理的房源数据保持一致。
- [ ] 前后端接口路径、字段、权限、提示保持一致。
