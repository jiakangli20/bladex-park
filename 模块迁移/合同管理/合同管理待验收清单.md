## 3. 验收状态总览

### 3.1 已校验过

- 退租审批 -> 房屋验收 -> 押金退还付款审批 主链路已做过一轮真实数据校验。
- 房屋验收通过后释放房源已校验。
- 退租列表页“加载即创建押金退款单”的副作用已修复。

### 3.2 已开发但未完整回归

- 合同审批主线
- 付款审批 / 开票申请回写账单
- 逾期提醒 / 律师函 / 通知书生成
- 原模板打印与下载
- 驳回后重提
- 多次发起场景
- 续租与补充协议归档

### 3.3 待重点核验

- `pay` 与 `invoice` 已拆成两个流程，需要分别核验入口、回写和打印文件。
- 原模板打印目前不能只验“能下载”，还要逐张验字段是否齐、版式是否稳。

## 6. 当前特别注意事项

1. `contract_sign` 现在已经切到 `contract`，这点和旧文档口径不同，验收时以当前部署为准。
2. `pay` 当前最新版本已经切成 `付款流程-付款申请单`，不再是旧的开票单。
3. `invoice` 已独立出来，后续“开票”和“付款”必须分开验。
4. 当前入驻审批实际应以 `entry + tenacy-in` 为准；如果后续还保留旧的 `tenant_entry-1` 外表流程，需要明确它是否继续使用，避免两个入驻流程并存造成混淆。
5. 退租列表页“查询即建单”的问题已修复，后面如果再验到押金退款单异常新增，优先排查是否有别的页面也在调用创建型接口。
